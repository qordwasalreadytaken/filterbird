Step-by-Step Flying Axe Test Instructions

Step 1: Open FilterBird

Navigate to http://localhost:8001

Step 2: Set Version to Path of Diablo

Click on "Version" in the menu and select "Path of Diablo" (not Project D2)

Step 3: Load the Filter

In Filter #1 text area, paste this single line:

ItemDisplay[(7tk OR 7bk OR 7ta) (MAG OR RARE OR NMAG) (!ETH AND !INF) FILTERLVL<3]: %GRAY% throw %TAN%[35k]

Step 4: Set Up Flying Axe

In the item selection area:

  • Group: weapon
  • Type: throwing weapon
  • Base: Flying Axe
  • Rarity: Regular (this is crucial!)

Step 5: Check FILTERLVL

Make sure FILTERLVL is set to 1 or 2 (anything less than 3)

Step 6: Expected Result

You should see: throw [35k]

What the Fix Does

Before the fix: Flying Axe defaulted to unique quality (UNI=true), which doesn't match (MAG OR RARE OR NMAG)

After the fix: Flying Axe defaults to normal quality (NMAG=true), which matches the condition

If It Still Doesn't Work

Possible issues:

  • Make sure you're using "Regular" rarity, not leaving it unset
  • Check that the version is set to Path of Diablo (not Project D2)
  • Verify FILTERLVL is less than 3
  • Make sure there are no typos in the filter line
  • Check browser console for JavaScript errors

Alternative Test - Try This Filter Line Instead

If the above doesn't work, try this simpler version:

ItemDisplay[7ta NMAG]: Flying Axe Test

This should definitely show "Flying Axe Test" if the fix is working.

Technical Details

The fix changes this code in custom_items.js line ~1813:

// BEFORE (broken):
} else {itemToCompare.UNI = true }

// AFTER (fixed):  
} else {
    // Default to normal quality (NMAG) for base items instead of unique
    itemToCompare.NMAG = true; 
    itemToCompare.always_id = true;
}
